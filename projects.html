Secure Photo Uploader for Healthcare Professionals

Industry Project · University of Tasmania · Client: HealthCare Software

As part of my final-year industry project, I worked with a small team to design and build a secure photo uploader for healthcare professionals. The goal was to give clinicians a safer way to capture and upload patient images from their mobile devices into a controlled cloud environment, instead of keeping sensitive photos on personal phones.

The solution combined a mobile application with a cloud backend using Microsoft Azure services for storage, identity and data management. My focus in the team was on security and testing: understanding the key risks, planning how to verify that the system behaved correctly and helping the team check that new features did not introduce unexpected issues.

My Role

  • Worked with the team to understand the client’s requirements around privacy, consent and secure handling of patient images.
  • Designed and executed test cases for sign-in, authorisation, photo capture and upload flows, including edge cases and failure scenarios.
  • Helped validate that images and related data were correctly stored and retrieved in the Azure environment according to the intended design.
  • Documented defects, retested fixes and contributed to progress updates and demonstrations for the client and supervisors.

What I Learned

  • How to think about security, privacy and usability together when building systems for real users in a sensitive domain.
  • Working in an Agile-style project with an external client, where communication and clear documentation are as important as the code itself.
  • Practical experience collaborating with others on a shared codebase and test plan, and adapting as requirements evolved.

Due to confidentiality, I do not share screenshots or technical implementation details from this project in my portfolio, but I am happy to discuss my role and approach in more detail during an interview.

Virtual Travel Journal

Personal / Coursework Project · Full-Stack Web Application

The Virtual Travel Journal is a web application that allows users to register, log in and record their travel experiences as journal entries, with an archive view for browsing previous trips. I originally built it as a small full-stack project using HTML, CSS, JavaScript, PHP and MySQL, handling both the front end and the server-side logic.

The current version of the site in my portfolio showcases the front-end implementation, including the layout, navigation and forms for creating and viewing entries. The original backend that handled authentication and data storage is no longer deployed, but the structure of the pages reflects how the system worked end-to-end.

Key Features

  • User registration and login forms designed to support account creation and authentication.
  • A create-entry page where users could add trip details, descriptions and images to their journal.
  • An archive page layout for browsing previous entries in a visually organised way.
  • Responsive styling to keep the site readable on different screen sizes.
